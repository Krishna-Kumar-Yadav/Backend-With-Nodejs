Versioning of packages or modules

Examples - 4.18.2   (express 4th version)

1st part -> 4
2nd part -> 18
3rd part -> 2

1st part (4) -> When there is major bug issues and need to update the package (5.18.2)

2nd part (18) -> When there is not major bug issue but the bug issue is recommended and need update (4.19.2)

3rd part(2) -> When there is minor bug issue not important but can be update(4.18.3)
